Quick Start

JavaScript
Python
cURL
import { Eternium } from 'eternium-sdk'

const client = new Eternium('etrn_your_key')

// Generate image — returns download URL when done
const img = await client.image('A futuristic city at sunset')
console.log(img.url)

// Generate video
const vid = await client.video('Drone flyover at night', { duration: 10, sound: true })
console.log(vid.url)

// Run a pipeline — 3 product shots, one call
const pack = await client.runPipeline('product-shot', 'White sneakers on marble')
pack.results.forEach(r => console.log(r.url))

// Check your usage
const usage = await client.getUsage()
console.log(`${usage.remaining} remaining this month`)
from eternium import Eternium

client = Eternium("etrn_your_key")

# Generate image — returns download URL when done
img = client.image("A futuristic city at sunset")
print(img["url"])

# Generate video
vid = client.video("Drone flyover at night", duration=10, sound=True)
print(vid["url"])

# Run a pipeline — 3 product shots, one call
pack = client.run_pipeline("product-shot", "White sneakers on marble")
for r in pack["results"]:
    print(r["url"])

# Check your usage
usage = client.get_usage()
print(f"{usage['remaining']} remaining this month")
# Generate an image
curl -X POST https://api.eternium.ai/v1/generate \
  -H "Content-Type: application/json" \
  -H "X-API-Key: etrn_your_key" \
  -d '{"model":"nano-banana-pro","prompt":"A futuristic city at sunset"}'

# Run a pipeline
curl -X POST https://api.eternium.ai/v1/pipelines/run \
  -H "Content-Type: application/json" \
  -H "X-API-Key: etrn_your_key" \
  -d '{"pipeline":"product-shot","prompt":"White sneakers on marble"}'

# Check task status
curl https://api.eternium.ai/v1/tasks/TASK_ID \
  -H "X-API-Key: etrn_your_key"

# Check your usage
curl https://api.eternium.ai/v1/usage \
  -H "X-API-Key: etrn_your_key"

Endpoints

POST/v1/generate

Create an image or video generation task.

Request Body

FieldTypeRequiredDescription
modelstringYesModel ID (see models above)
promptstringYesText description of what to generate
cachebooleanNoEnable prompt caching (default: true). Agents save money here.
callback_urlstringNoWebhook URL for completion notification
image_urlsstring[]NoReference images (image-to-image/video)
aspect_ratiostringNo"16:9", "9:16", "1:1" (default varies)
durationnumberNoVideo length in seconds (3-15)
modestringNo"std" or "pro" (video quality)
soundbooleanNoInclude audio (video only)
resolutionstringNo"1K"/"4K" (image), "720p"/"1080p" (video)

Response

{
  "code": 200,
  "msg": "success",
  "data": { "taskId": "task_abc123" },
  "_cost": "$0.04",
  "_budget_remaining": "$61.96",
  "_cached": false
}

Smart Caching

Every generation request is cached by default. If an agent sends the same prompt + model + parameters twice, the second request returns instantly at zero cost. Cache TTL is 1 hour.

Disable per-request with "cache": false in the request body.

Cached responses include "_cached": true and "_saved": "$0.04" so agents can track their savings.
